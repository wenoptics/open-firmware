# Scribit Pen Carousel Reference

---

## Layout

The carousel has **4 pen slots** at fixed Z angles (measured in degrees from
the Hall sensor reference):

| Pen | Z degrees |
|-----|-----------|
| 1   | 89        |
| 2   | 161       |
| 3   | 233       |
| 4   | 305       |

```python
# .reference/lqu/src/scribit_svg_to_gcode.py:44
PEN_SLOTS_Z: Dict[int, int] = {1: 89, 2: 161, 3: 233, 4: 305}
```

---

## Homing the Carousel (G77)

Before any pen selection G77 must run to establish Z=0 at the Hall sensor
magnet reference.

```gcode
G77          ; home carousel via Hall sensor
G92 Z-56     ; set Z reference: "after homing, you are at -56°"
```

**Why `G92 Z-56`?** After G77 the internal Z position register is 0 (the
firmware sets it in `home_cylinder()` and `faultyMagnetCalibration()`). But
the physical carousel is not at the "pen 1" slot. The offset `-56` has been
determined empirically to produce correct absolute Z values for the four pen
slots.

```python
# .reference/lqu/src/scribit_svg_to_gcode.py:46-47
Z_AFTER_G77 = -56.0
```

After homing, the carousel state tracks `z = -56.0`.

---

## Carousel Rotation Direction: CCW Only

The carousel must rotate **counter-clockwise only** to avoid cable tangling.
When selecting a pen slot whose absolute Z degree value is lower than the
current carousel position, wrap by adding 360°:

```python
# .reference/lqu/src/scribit_svg_to_gcode.py:56-64
def ccw_only_target(current_z, slot_z):
    target = float(slot_z)
    while target < current_z:
        target += 360.0
    return target
```

Example: if current Z is 305 (pen 4) and you want pen 1 (89°), emit
`G1 Z449.000` (= 89 + 360), not `G1 Z89`.

## Selecting a Pen (G1 Z in Absolute Mode)

Pen selection is a Z move in **absolute** mode:

```gcode
G90                         ; switch to absolute
G1 Z<target_degrees> F600   ; rotate carousel to pen slot
G91                         ; return to incremental
```

The firmware's `getSelectedPen()` in `g101.h` identifies which pen is active
by comparing `mechanics.destination[Z_AXIS]` to the integer slot values:

```cpp
// Firmware/MK4duo/src/core/commands/gcode/scribit/g101.h:9-24
if(mechanics.destination[Z_AXIS] == 89)       l_retVal = 0; // pen 1
else if(mechanics.destination[Z_AXIS] == 161) l_retVal = 1; // pen 2
else if(mechanics.destination[Z_AXIS] == 233) l_retVal = 2; // pen 3
else if(mechanics.destination[Z_AXIS] == 305) l_retVal = 3; // pen 4
```

**Critical constraint:** This comparison is an exact float equality check
against the base slot values 89, 161, 233, 305. Once CCW wrapping adds 360
(e.g. Z = 449 for pen 1 on the second visit), `getSelectedPen()` returns
error (-1) and **G101 falls back to a fixed 30° engagement** rather than the
pen-calibrated angle. This is the primary cause of incorrect pen engagement
when using G-code generated by tools that wrap Z past 360.

**Workaround:** After each pen-up, re-home the carousel with G77 + `G92 Z-56`
before the next pen selection, resetting Z to base values. This adds latency
but guarantees correct slot recognition. Alternatively, keep drawings to a
single pen so Z never exceeds the first slot's base value.

## Engaging the Pen (G101 — Pen Down)

`G101` moves the carousel Z down by the pen-calibrated engagement angle
(`g_timeLimit[pen]`, default 30°). For a reliable latch, issue it three times:

```gcode
G101
G101
G101
```

```python
# .reference/lqu/src/scribit_svg_to_gcode.py:136-138
def gcode_pen_down():
    return ["G101", "G101", "G101"]
```

Each `G101` call:
1. Calls `getSelectedPen()` to identify which pen is active.
2. Subtracts `g_timeLimit[pen]` from `destination[Z_AXIS]` and executes the
   move.

If the pen cannot be identified (Z not at a base slot value), it falls back to
subtracting 30° regardless.

```cpp
// Firmware/MK4duo/src/core/commands/gcode/scribit/g101.h:60-67
if(l_selectedPen >= 0 && l_selectedPen <= 3)
    mechanics.destination[Z_AXIS] -= g_timeLimit[l_selectedPen];
else
    mechanics.destination[Z_AXIS] -= 30;  // fallback
```

## Retracting the Pen (Pen Up)

Pen-up is simply re-selecting the same pen slot (returning to the slot's Z
angle, CCW-only):

```python
# .reference/lqu/src/scribit_svg_to_gcode.py:140-142
def gcode_pen_up(pen, f_z, st):
    return gcode_pen_select_ccw(pen, f_z, st)
```

## Pen-Engagement Calibration (G100)

G100 calibrates the per-pen engagement angle (`g_timeLimit[pen]`) using the
IMU gyroscope. It must be run with the carousel already positioned at the
correct pen slot (i.e., after `G77` + pen selection). Run it as part of a
factory or first-use setup sequence; the calibrated values are stored in RAM
and are lost on reset.

## Known Bug: CCW Wrap Breaks Pen Identification

**Symptom:** When `scribit_svg_to_gcode.py` generates G-code with more than one
pen selection cycle (pen 1 → pen 4 → pen 1 again), subsequent pen engagements
use the wrong depth or do not engage at all.

**Root cause:**

The `getSelectedPen()` function identifies the active pen by comparing
`destination[Z_AXIS]` to the **base slot values** 89, 161, 233, 305
using exact float equality:

```cpp
// Firmware/MK4duo/src/core/commands/gcode/scribit/g101.h:9-24
if(mechanics.destination[Z_AXIS] == 89)  ...
```

The CCW-only rotation rule causes Z to accumulate past 360 on revisits. After
pen 4 (Z=305), pen 1 requires Z=449 (= 89 + 360). The firmware receives
`G1 Z449.000`, sets `destination[Z_AXIS] = 449.0`, then G101 cannot match any
slot and falls back to a fixed 30° engagement that may not press the pen far
enough to make contact.

**Workaround options:**

1. **Re-home after every pen-up.** Insert `G77` + `G92 Z-56` before each new
   pen selection. This resets Z to -56, so the next slot target stays at the
   base value (e.g. 89). Adds ~5 s per pen change.

2. **Single-pen drawings.** Only use one pen per G-code file. Z never wraps.

3. **Firmware fix (recommended long term).** Change `getSelectedPen()` to
   normalize the Z value modulo 360 before comparison:
   ```cpp
   float z_mod = fmod(mechanics.destination[Z_AXIS], 360.0f);
   if (z_mod < 0) z_mod += 360.0f;
   if (z_mod == 89) ...
   ```
   This would allow arbitrary CCW wrapping without breaking pen identification.
